### The Problem

Adam Attar runs a cleaning, caretaker, moving, and gardening service company in Kirn, Germany. He needed a professional online presence that could:

- **Generate leads** — visitors should be able to contact him or apply for jobs directly
- **Build trust** — showcase completed projects, list certifications, display customer reviews
- **Rank on Google** — appear in local search results for "Reinigungsservice Kirn" and similar queries
- **Be privacy-compliant** — handle contact form data and cookies under German DSGVO law
- **Stay simple** — the site had to be maintainable by someone who isn't a developer

### Solution

I built a fully static (but server-rendered where needed) website using Astro 5 with Tailwind CSS v4, deployed on Vercel. The site combines a polished marketing frontend with practical backend features like contact forms, job applications with file upload, and a project gallery.

### Why This Stack

**Astro** was the obvious choice — content-heavy sites are what it's built for. Zero JavaScript by default means the homepage loads fast even on slow connections. When I needed server-side features (form endpoints), Astro's hybrid SSR handled it without a separate backend.

**Tailwind CSS v4** because the design required consistent branding (a distinctive orange `#ff5f00`) across many pages, and Tailwind's utility-first approach made it easy to iterate on the layout without writing custom CSS.

**Vercel** for deployment — the platform handles SSL, CDN, serverless functions, and environment variables seamlessly. The built-in Analytics and Speed Insights were a bonus.

**Nodemailer via Gmail SMTP** was the simplest way to send contact inquiries and job applications as emails. No third-party service, no monthly fee — just a Gmail App Password.

### Key Features

#### 1. Service Pages with Structured Data

Each of the four services (Reinigung, Hausmeister, Umzug, Garten) has a dedicated detail page with sub-services, a 4-step process walkthrough, FAQ accordions, and service area information. Every page includes schema.org structured data (`Service`, `FAQPage`, `BreadcrumbList`).

#### 2. GDPR-Compliant Cookie Consent

Full cookie consent system with Google Consent Mode v2. The banner appears on first visit with Accept / Reject / Settings options. The settings modal lets users toggle Analytics (GA4) and Marketing (Google Ads) consent independently. All preferences are stored in `localStorage` with versioning for future updates.

#### 3. Contact & Application Forms

Two server-side API endpoints:

- **`/api/contact`** — validates name, email, phone, and message. Sends an internal notification to the company and an auto-reply confirmation to the customer. Includes honeypot anti-spam and server-side validation.
- **`/api/apply`** — accepts job applications with optional CV upload (PDF/DOC, max 5MB). Sanitizes filenames, validates file types, and sends everything via email with the CV as an attachment.

#### 4. Project Gallery with Lightbox

A reusable `ProjectGallery` component that displays project cards in a grid. Clicking a card opens a full-screen lightbox with Swiper carousel navigation. Each project is tagged with its service category and photo count.

#### 5. Comprehensive SEO

JSON-LD structured data across every page: `Organization`, `LocalBusiness`, `WebSite`, `Service`, `FAQPage`, `JobPosting`, `BreadcrumbList`, `ItemList`. Every page has Open Graph and Twitter Card meta tags. The sitemap is auto-generated by `@astrojs/sitemap`.

### Challenges & Solutions

#### CSP Headers with Inline Scripts

Vercel applies strict Content-Security-Policy headers by default. The cookie consent script and Google Tag Manager both use inline scripts, which were blocked. I had to configure `script-src` nonces and `'unsafe-inline'` for the GTM script in `vercel.json`.

#### Gmail App Passwords

Setting up Nodemailer with Gmail required an App Password, but Google's account security settings have changed. The `.env.local` file now includes step-by-step instructions for generating one.

#### Image Optimization for the Hero Video

The hero section uses a background video (`main.mp4`) with a poster image (`main-poster.jpg`). Getting the video to load quickly without layout shift required explicit aspect ratio containers and the poster attribute.

#### Tailwind v4 Migration Gotchas

Tailwind v4 removes the `tailwind.config.js` file entirely in favor of CSS-first configuration. The `@theme` directive replaces `theme.extend` in the config. Some utility class names changed — I had to audit the entire codebase when upgrading.

### Lessons Learned

Astro 5 paired with Tailwind CSS v4 is an excellent stack for content-driven business websites. The zero-JS-by-default approach means fast load times, and the hybrid SSR handles forms and dynamic endpoints without needing a separate backend.

The biggest takeaway was the importance of structured data for local SEO. For a service business like this, the schema.org markup probably does more for visibility than any visual design choice.

I also learned that German GDPR compliance adds significant complexity — the cookie consent system alone required careful timing between Google Tag Manager, Consent Mode, and the user interaction flow. It's not something you can bolt on at the end.
